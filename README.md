## 🎯 Propósito del Repositorio

Este repositorio fue diseñado como un complemento académico integral para la materia **Programación Orientada a Objetos (POO 2026)**, impartida en las carreras de **Analista de Sistemas** y **Licenciatura en Sistemas** de la **Universidad Nacional de La Patagonia Austral (UARG)**.

### Objetivo Educativo

El material que vas a encontrar acá tiene un propósito claro: **enriquecer tu experiencia de aprendizaje** más allá de lo visto en clases teóricas y prácticas. No pretendo que este material sea un sustituto de la bibliografía recomendada ni de la enseñanza en aula, sino un complemento, **un puente entre la teoría y la práctica** que:

✨ **Complementa** los conceptos teóricos con ejercicios prácticos contextualizados  
✨ **Expande** la bibliografía oficial con explicaciones adicionales y ejemplos concretos  
✨ **Acompaña** tu proceso de aprendizaje con materiales pensados desde la perspectiva del estudiante  
✨ **Fortalece** la comprensión de POO mediante proyectos integrados y documentación detallada

## 🎓 Estructura de Cada Proyecto

Cada proyecto en este repositorio sigue una estructura consistente:

### 📄 **enunciado/**
- Especificación completa del problema a resolver
- Requisitos y expectativas
- Ejemplos de entrada/salida cuando aplique

### 📖 **documentacion_soporte/**
- Material complementario sobre conceptos clave
- Explicaciones previas a la solución
- Documentación teórica que acompaña la práctica
- No reemplaza la bibliografía, sino que la amplía

### 💻 **implementacion/**
- Código Java completamente comentado
- Solución bien estructurada del problema
- Ejemplos de buenas prácticas
- Código documentado con JavaDoc

### 📋 **README.md** (específico de cada proyecto)
- Descripción detallada del proyecto
- Cómo compilar y ejecutar
- Guía de uso
- Análisis de la solución

## 🚀 Cómo Usar Este Repositorio

### Para Estudiantes

1. **Entiende el enunciado** del trabajo/diagnóstico
   - Lee el archivo `enunciado.md` en la carpeta del proyecto
   
2. **Consulta el material de soporte** si necesitas contexto
   - Documentación teorica complementaria
   - Explicaciones de conceptos clave
   - Ejemplos progresivos
   
3. **Estudia la solución implementada**
   - Lee el código comentado
   - Ejecuta y prueba el programa
   - Analiza las decisiones de diseño
   
4. **Practica y experimenta**
   - Modifica el código
   - Agrega nuevas funcionalidades
   - Resuelve variaciones del problema

5. **Comparte y consulta**
   - Discute con compañeros
   - Plantea dudas a docentes
   - Comparte mejoras o sugerencias

### Para Docentes

- Utiliza este repositorio como **material de referencia complementario** en tus clases
- Los proyectos sirven como **ejemplos prácticos** de los conceptos teóricos
- La documentación complementaria puede reforzar tópicos específicos
- El código bien documentado es ideal para **análisis colaborativo en clase**
- Puedes proponer variaciones/extensiones de los problemas como desafíos adicionales

## 🔧 Requisitos Técnicos (Generales)

- **Java JDK**: Versión 8 o superior
- **Sistema Operativo**: Linux (OpenSUSE en lo posible), Windows o macOS
- **Terminal/Consola**: Para compilar y ejecutar
- **Editor de Código** (recomendado):
  - VS Code con extensión "Extension Pack for Java"
  - NetBeans

> Nota: Cada proyecto puede tener requisitos específicos adicionales documentados en su README

## 🎯 Objetivos de Aprendizaje General

Al trabajar en serio con este material a lo largo de la cursada, deberías ser capaz de:

- ✅ Entender los pilares de la Programación Orientada a Objetos
- ✅ Modelar problemas del mundo real en estructuras de datos
- ✅ Escribir código limpio, modular y bien documentado
- ✅ Aplicar patrones de diseño en soluciones prácticas
- ✅ Leer, analizar y extender código Java existente
- ✅ Pensar de forma orientada a objetos en problemas complejos
- ✅ Colaborar y comunicar ideas técnicas efectivamente

##💡 Filosofía del Material

Este repositorio está construido sobre principios educativos fundamentales:

1. **Claridad primero**: El código y la documentación deben ser fáciles de entender
2. **Contextualización**: Los conceptos se enseñan mediante problemas reales
3. **Progresividad**: De lo simple a lo complejo, paso a paso
4. **Acompañamiento**: El material está diseñado desde la perspectiva del estudiante
5. **Practicidad**: Todo lo que se enseña tiene una aplicación concreta
6. **Contención**: Sabemos que aprender a programar es desafiante; estamos aquí para apoyarte

## 📚 Recomendaciones de Estudio

- **No memorices el código**: **Entiéndelo**. Lee, analiza, copia a mano, experimenta.
- **Practica modificando**: Cambia valores, agrega métodos, prueba variaciones.
- **Haz preguntas**: Consulta con compañeros y docentes sobre dudas surgidas.
- **Combina recursos**: Usa esto junto con tus apuntes, libros y prácticas en clase.
- **Dedica tiempo**: La programación se aprende **haciendo**, no solo leyendo.
- **Forma un grupo de estudio**: Discutir con otros es una de las mejores formas de aprender.
